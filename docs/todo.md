请帮我开发一个“长期陪伴型 AI 平台”的网站 MVP。

技术栈请使用 Next.js + TypeScript + Tailwind CSS，优先做前端可运行原型，可使用本地 mock 数据，不要求真实后端。页面要能完整跳转，交互要尽量真实，结构要方便后续接接口。并且我需要网页端和移动端都正常美观的显示。

产品定位：
这是一个面向长期陪伴型 AI 的平台，用户可以管理自己的账号、充值、调试智能体提示词、查看不同记忆体的记忆，并在社区上传和使用其他人的智能体模板。

请实现 6 个页面：

1. 登录页
- 用户登录入口
- 包含手机号 / 邮箱输入、密码输入、登录按钮
- 可有注册、忘记密码入口，但不需要真实实现
- 登录后进入用户主页

2. 用户主页
- 展示用户基础信息：头像、昵称、账号 ID、会员状态、余额
- 展示快捷入口：提示词调试、记忆查看、社区、充值
- 展示用户拥有的记忆体列表
- 每个记忆体显示名称、简介、最近更新时间
- 支持切换当前记忆体

3. 充值页
- 展示当前余额和会员状态
- 展示多个充值套餐卡片
- 支持选择套餐
- 点击充值后展示模拟支付成功状态
- 充值是账号级功能，不与记忆体绑定

4. 提示词调试页
- 顶部有当前记忆体选择器，可切换记忆体
- 左侧为提示词编辑区
- 支持输入系统提示词 / 人设提示词
- 支持从社区模板导入
- 右侧为调试聊天窗口
- 用户输入测试消息后，AI 返回 mock 回复
- 显示当前使用的模板名称、模型参数、记忆体名称
- 支持保存当前提示词到当前记忆体
- 注意：提示词调试必须与当前记忆体绑定

5. 社区页
- 展示智能体模板社区
- 模板卡片包含：模板名称、作者、简介、标签、使用次数、审核状态
- 支持搜索、标签筛选、排序
- 支持上传模板按钮
- 上传模板弹窗包含：模板名称、简介、标签、提示词内容
- 用户上传后进入“待审核”状态
- 公开社区只展示审核通过的模板
- 模板详情页或弹窗中支持“导入到提示词调试”
- 社区是账号级公共功能，不与记忆体绑定

6. 记忆查看页
- 顶部有当前记忆体选择器，可切换记忆体
- 支持图谱形式查看，上面是一个条状作为日期，然后每个日期对应一个记忆的连通图显示，但只能查看不能修改，显示要美观。
- 初版只允许查看，不开放编辑
- 可以展示“编辑功能暂未开放”的灰色按钮或提示
- 预留未来日历视图、日记视图入口，但不需要实现

数据结构请至少包含：
- User
- MemorySpace / 记忆体
- PromptTemplate / 智能体模板
- MemoryItem / 记忆条目
- RechargePlan
- AuditStatus

设计要求：
- 整体风格温柔、干净、有陪伴感，但不要做成花哨营销页
- 首屏应该直接是产品功能，不要做大段 landing page
- 使用清晰的侧边栏或顶部导航
- 页面之间跳转顺畅
- 卡片圆角不要太大，控制在 8px 左右
- 信息密度适中，适合真实后台产品使用
- 不要使用大面积紫色渐变
- 不要使用空洞宣传文案，重点做可操作界面

交互要求：
- 登录后进入用户主页
- 用户可在提示词调试页和记忆查看页切换记忆体
- 从社区导入模板后，跳转到提示词调试页，并把模板内容填入编辑器
- 上传模板后显示待发布或待处理状态
- 记忆查看页的编辑按钮不可用，并提示“记忆编辑功能暂未开放”
